Story-Sim 工作流全景图 v6 — 2026-06-02 (简化版)

事件调度 → 角色决策 → 世界模拟 → 叙事编排 | 三组件已移除: 弧状态机 / 伏笔 buffer / auto_event

主流程拓扑

Step 1: 世界时钟 + Occupancy

1_get_next_world_time() + Occupancy Model
纯规则
▶
取所有角色 personal_schedule[0] 的最早时间点 → 世界时钟
精确匹配 day/hour/minute → 活跃角色列表
弹出事件（丢弃同角色同时刻的重复事件），返回 {cid: {description, duration, importance}}
清理过期 Occupancy → 写入新占用（busy_until = 当前时间 + duration, locked_location 由 activity 关键词推断）
过期占用角色自动写入"自由活动"事件，时间 clamp 到当前世界时钟

Step 2: 角色意图生成

2_generate_character_intent()
LLM xNasync
▶
占用角色跳过 LLM，用 locked_location + activity 占位（importance 截断为 6）
记忆检索：_extract_query_keywords() 从 location/mood/actions/contacts/plan 提取关键词，检索 5 条
构建 Scratchpad + 关系摘要 + affected_events 差异化上下文
LLM 并行调用（async with Semaphore(6)），json_mode=True，null 防御清洗
next_event 写入 personal_schedule：绝对时间 + 防倒退校验 + 总分钟数跨日判断
Prompt
System: 角色模拟器，根据设定/情境/记忆/关系/事件/弧光决定意图 - destination 约束为标准地点列表 (18个) - next_event 时间: {"next_event_day": N, "next_event_hour": H, "next_event_minute": M} User: 请按照要求输出 JSON。next_event 使用绝对时间，必须晚于当前时间。

Step 3: 世界模拟

3detect_encounters() + generate_dialogue()
碰撞:规则对话:LLM
▶
按 destination 分组（含非活跃角色：同位置的 character_states 也纳入）
意图交叉校验："找/见/等/约 + 角色名" → 目标不在同地 → 标记失败 + 降级 importance
关系阈值判定：trust>70/affection>70/trust<30/affection<30 → DIALOGUE；trust≥40/affection≥40 → OBSERVATION；其他 → PASSING
DIALOGUE 调用 LLM 生成对话（并行），输出 dialogue + outcome + relationship_delta
显式检查 gather 结果，异常记录 error + 兜底 outcome
关系Δ

dialogue_data 存入 Encounter 模型字段；relationship_delta 缺失时从 outcome 启发式推断（正面词→+2~3, 负面词→-3~-2）；Step 4 中调用 relationship.update_relationship() 应用增量

Step 4: 状态更新 + Skip 判定

4update + memory + skip
纯规则
▶
遍历 active_interactions：更新 last_action/location/mood/condition，拼接记忆描述
应用 relationship_delta：遍历 dialogue_encounters → update_relationship(trust_delta, affection_delta)
记忆写入：加权评分淘汰（importance×10 + step×0.5 + reflection_bonus 30）
Skip 判定（简化）

max_importance < 4 AND !has_encounters AND !has_world_events AND consecutive_skipped < 5 → skip。仅统计活跃角色参与的碰撞。skip 时仍写入 timeline（标记 skipped=True）和 world_snapshot，但不生成叙事。

Step 5: 叙事编排

5_generate_narrative_with_foreshadowing()
LLM
▶
故事摘要：最近 20 步叙事，每条截 150 字
角色卡构建 + 场景分组 + 时间线指导（同一 LLM 调用，全局视角）
LLM 生成叙事 + narrative_step_count 递增
Prompt 规则

人设硬约束(fear/secret/background不可违反) | 去重(相似场景聚焦新细节) | telepathy区分(脑内传音用破折号/叙事标记) | 同时性(相邻场景合并/视角切换)

反思机制

R_run_reflections()
LLM xN
▶
narrative_step_count % 3 == 0 时触发
取最近 6 步非反思记忆 → LLM 生成洞察
覆盖所有角色（非仅活跃），反思记忆 importance floor=7
